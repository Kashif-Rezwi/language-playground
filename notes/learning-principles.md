# Learning Principles

This repo is designed for practical, consistent learning rather than passive note collection.

## Core Rules

- Code every day before expanding notes.
- Keep each day focused on one main concept.
- Prefer small real-world scripts over isolated syntax examples.
- Build projects that reuse earlier lessons.
- Track confidence honestly, including blockers.
- Keep the repo clean enough that future review feels easy.

## Practice Loop

1. Read the day's goals.
2. Run the provided examples.
3. Solve exercises without copying first.
4. Refactor the code once it works.
5. Record the key takeaway and confidence score.

## AI/ML Preparation Mindset

Python for AI/ML is not only about libraries. The foundation is:

- clean data handling
- reliable functions
- readable project structure
- debugging confidence
- comfort with files, CSVs, and structured records
- basic testing and reproducibility

Month 1 should make these basics feel natural before moving deeper into machine learning.
